5. COPC Hierarchy Entry & 옥트리 개념도 core idea

Hierarchy Entry 구조 (32 bytes / entry)
Key: Level, X, Y, Z4×4 B = 16 B
Offset (파일 내 청크 위치)8 B
Byte Size (청크 압축 크기)4 B
Point Count4 B

Point Count가 음수이면 "이 항목은 실제 포인트가 아니라 하위 Hierarchy 페이지를 가리키는 포인터"라는 의미입니다. 이렇게 트리를 재귀적으로 따라가며 필요한 해상도까지만 읽을 수 있습니다.

Root (Level 0) — 전체 영역, 가장 낮은 해상도
├─ Level 1 — 8개 하위 옥트리 노드 (x2 해상도)
├─ Level 2 — 필요한 영역만 세분화
└─ Level N — 요청한 해상도/영역에 도달하면 중단

클라이언트는 헤더 + Hierarchy VLR만 먼저 내려받아 트리를 파악한 뒤, 필요한 노드의 (Offset, Byte Size)만 골라 HTTP Range Request로 그 부분만 스트리밍합니다. 이것이 COPC가 "Cloud Optimized"인 이유입니다.
